# 1Panel WAF 和网站页面集合

这是一个为 1Panel 面板设计的 WAF（Web 应用防火墙）和网站相关页面集合，提供了一系列专业的错误提示和状态页面。

## 项目结构

项目分为两个版本（1.0 和 2.0），每个版本包含以下目录：

- **WAF/**: 包含安全相关页面，用于显示 WAF 拦截信息
- **网站/**: 包含网站相关页面，用于显示网站访问错误和状态
- **其他/**: 包含维护页面等其他类型的页面

### 2.0 版本文件列表

#### WAF 目录
- `IP 黑名单.html` - IP 被列入黑名单时的提示页面
- `地区限制.html` - 地区访问限制提示页面
- `禁止访问 (403).html` - 403 禁止访问提示页面
- `网站不存在.html` - 网站不存在提示页面

#### 网站目录
- `网站 404 错误页.html` - 404 页面不存在错误提示
- `网站不存在页.html` - 网站不存在提示页面
- `网站停用页.html` - 网站停用提示页面
- `网站默认站点.html` - 网站默认站点页面

#### 其他目录
- `dnslj.html` - DNS 解析相关页面
- `网站维护.html` - 网站维护中提示页面

### 1.0 版本文件列表

1.0 版本包含与 2.0 版本类似的文件，但可能在设计和功能上有所不同。

## 页面特点

### 设计风格
- 采用深色终端风格设计，具有科技感
- 统一的视觉元素和交互体验
- 响应式布局，支持移动设备
- 包含平滑的动画效果

### 功能特性
- **唯一 ID 生成**：每个页面加载时会生成唯一 ID，便于问题追踪
- **弹出窗口**：提供详细信息的弹出窗口
- **交互按钮**：包括申诉、联系管理员、刷新页面等功能
- **响应式设计**：适配不同屏幕尺寸
- **统一的联系方式**：提供邮箱联系渠道

### 技术实现
- 纯 HTML/CSS/JavaScript 实现，无需外部依赖
- 使用内联 CSS 和 JavaScript，确保页面独立性
- 响应式设计，使用媒体查询适配不同设备
- 模块化结构，便于维护和扩展

## 使用方法

1. 选择适合的版本（1.0 或 2.0）
2. 根据需要将相应的 HTML 文件复制到您的 1Panel 面板相关目录
3. 根据实际情况修改页面中的联系邮箱和其他信息
4. 在 1Panel 面板中配置相应的错误页面指向

## 自定义指南

### 修改联系信息
1. 打开相应的 HTML 文件
2. 找到包含邮箱地址的部分（通常在 `appeal-email` 或 `maintenance-info` 类中）
3. 将邮箱地址修改为您的实际联系邮箱

### 修改页面内容
1. 打开相应的 HTML 文件
2. 修改页面中的文本内容，如错误信息、维护信息等
3. 可以根据需要调整样式和布局

### 修改页面标题和图标
1. 打开相应的 HTML 文件
2. 修改 `<title>` 标签中的内容
3. 修改 `<link rel="icon" href="...">` 中的图标 URL

## 版本说明

- **1.0 版本**：基础版本，包含核心功能
- **2.0 版本**：增强版本，可能包含更多功能和改进的设计

## 注意事项

- 所有页面均使用内联 CSS 和 JavaScript，确保页面独立性
- 页面中的图标使用了外部 CDN 链接，确保网络连接正常
- 如需完全离线使用，可将图标下载到本地并修改相应路径
